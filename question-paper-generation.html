Question Paper Generation


Web Application - UI-UX

With the onset of computer based technology there have been evolutionary changes in many areas of our professional environments. Most strangely e-Education and e-Learning is highly influenced. There is a shift from manual to automated systems for different aspects of education system. At every stage/level of education some admission procedure is present. The fundamental to this admission procedure is a Question paper.

However the main problem is, a low quality of papers generated and teachers need to invest a lot of time and energy in composing question papers.

CURRENT PROCESS

There is a composition for every type of paper which is considered while making the Question Paper.

For a particular exam, the faculty marks all the questions in the material according to the composition and the CEs copy paste the questions and it is verified for twice before it is sent for final printing.

There is a target date for every exam by which the QP should be prepared and sent for printing. If they fail doing so, the printing won’t happen and the exam won’t happen.

PROPOSED MODEL

WORKING


Admin/Employee Login
Admin/Teacher will be provided with the Employee id and Password. After successfull login, Admin/Teacher can select questions from the question bank. And this questions will be stored in the database in the form of question bank.

Previously Generated Question Paper
It will show the previously generated papers, also Teacher can duplicate previous papers.

Create New Question Paper
To create new question paper, select class, batch, exam type and exam date. And select subject, chapters and topics. Add questions from it.

Generate and Print Question Paper
Once all questions are added, Teachers can format the question paper such as they can change the fonts, color, border etc. and can print the paper.
